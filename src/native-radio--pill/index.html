Styled Native Radio Pills

Pattern to restyle a radio button group into a selection pill.

To style radio buttons into a pill-like component requires a specific markup pattern: 


<fieldset class="fieldset">
  <legend>Group Title</legend>

  <div class="r-group">
    <input type="radio" id="r1" name="radios" checked>
    <label for="r1">Option 1</label>

    <input type="radio" id="r2" name="radios">
    <label for="r2">Option 2</label>
  </div>
</fieldset>

As radio buttons do not have a native way to revert to a completely unchecked state, without usage of a form reset button or a user performing a hard refresh with their browser, there should always be a safe default option checked, so a user doesn't have to reset a form to remove an erroneous radio button selection.

The input that is presently checked will trigger the CSS selector to change the style of its associated sibling label, to visually indicate it is the selected radio button. As the radio buttons are only visually hidden, they are still accessible to screen readers and keyboard users. Screen readers will announce the state of the selected radio button. A specific selector, utilizing the last label's :after pseudo-element, will create an outline around the entire component to indicate it has keyboard focus.

Note: It is considered a WCAG 3.2.2, On Input, violation to change state of an interface when interacting with an input that is not announced as a "button" (e.g. button or input with a type value of button, reset or submit, for example. Radio buttons are meant to indicate a group of single choice options, typically part of a larger form area of a document.

This pattern may be perfectly acceptable to use in the context of a form, or as a UI changing mechanism as long as activation of a "confirmation" or "submit" button accompanies this component. Otherwise, a grouping of button elements, that are aware of each other's state, and utilize aria-pressed to indicate their own present state, or other type of ARIA pattern may be more appropriate to use as an instant document state-changing component.
