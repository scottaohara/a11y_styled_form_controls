Pattern Details

When customizing any form control, it must remain accessible to assistive technology and keyboard users. To keep this custom pattern accessible, a visually hidden radio form control is the cornerstone of the custom style.

Without the visually hidden radio button, all radio functionality, including keyboard focus and appropriate announcements to screen readers, would need to be rewritten using JavaScript and necessary ARIA attributes. This would significantly inflate the development effort needed to code a custom radio button.

The appearance of the custom radio is determined by the state of the native radio button. :checked, and :focus states, along with the form control's disabled attribute all serve as hooks to visually change the styling of the custom radio button.

This pattern use one of two markup approaches:

The first pattern

Pattern Markup: Visually Hidden Style 
<div class="c-radio">
  <input type="radio" name="my_radios" id="unique_id">
  <label for="unique_id">
    Label text here
  </label>
</div>

This implementation relies on the wrapping div.c-radio container, whose primary purpose is to act as a styling hook. It utilizes position: relative; to keep absolutely positioned child and pseudo elements within the confines of the component.

The second pattern

Pattern Markup: Restyled Radio 
<label class="n-radio-label">
  <input type="radio" name="n_radio" id="native_r2">
  Label
</label>

With this implementation, the form control's label :before and :after pseudo elements are used to create a visual "pressed radio button" effect that can be styled via CSS.
