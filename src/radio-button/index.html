These custom radio button styles adhere to one of the two following markup patterns:

The first pattern

<!-- Visually Hidden Radio Style -->
<div class="c-radio">
  <input type="radio" name="my_radios" id="unique_id">
  <label for="unique_id">
	Label text here
  </label>
</div>

The .c-radio container largely serves as the styling hook, and utilizes position: relative; to keep absolutely positioned child and pseudo elements relative to the container.

The actual native radio button is visually hidden from view. While visually hidden, it is essential for it to remain accessible to screen readers and keyboard focus. Otherwise all the native radio button functionality would need to be rewritten in JavaScript.

The label's :before and :after pseudo elements are used to recreated the visible radio button.

The styling of the custom radio button is determined by the state of the native form control. The :checked, disabled, and :focus radio button states will all be reflected in the custom style.

The second pattern

<!-- Restyled Radio -->
<label class="n-radio-label">
  <input type="radio" name="n_radio" id="native_r2">
  Label
</label>

The second pattern uses the label to wrap the radio button, but this pattern isn't reliant on it as part of the setup. Instead, the radio button is restyled, having its native styling completely removed by appearance: none. It is then restyled largely with CSS borders and box-shadows.
