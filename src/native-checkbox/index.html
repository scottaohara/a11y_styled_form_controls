These custom checkbox styles require a specific markup pattern: 

<div class="c-cb">
  <input type="checkbox" name="c_input" id="c_input_1" checked>
  <label for="c_input_1">
    Option 1
  </label>
</div>

The .c-cb container largely serves as the styling hook, and utilizes position: relative; to keep absolutely positioned child elements and pseudo elements, within the container.

The actual native checkbox is visually hidden but remains accessible to screen reader's virtual cursors and standard keyboard focus with the tab key. Keeping the checkbox accessible to screen readers and keyboard users is essential in continuing to utilize the native functionality of a checkbox. Otherwise all checkbox functionality would need to be rewritten with JavaScript and appropriate ARIA attributes, which unnecessarily balloons the development effort needed to code a custom checkbox style.

The label's :before and :after pseudo elements are used to recreated the visible checkbox and "check" element.

The styling of the custom checkbox is determined by the state of the actual checkbox. :checked, and :focus states, along with the disabled attribute on the input will all visually change the styling of the custom checkbox.
